# Документация

Документация хранится в `docs/` и собирается через MkDocs Material.

## Локальный запуск

```powershell
pip install -r requirements-docs.txt
mkdocs serve
```

## Сборка

```powershell
mkdocs build
```

## Правила

- Важные архитектурные решения фиксируются в `docs/architecture/adr/`.
- README в корне должен оставаться коротким входом в проект.
- Подробные инструкции переносим в `docs/`.
- Документация меняется вместе с кодом в том же pull request, если изменение
  влияет на поведение, архитектуру или процесс разработки.
