# ADR 0001: FastAPI stack

## Статус

Принято.

## Контекст

Проект уже начат на FastAPI, а предыдущие версии показали, что доменная часть
хорошо ложится на API-first backend. Нужно сохранить FastAPI и стандартизировать
сопутствующий стек.

## Решение

Используем:

- FastAPI для backend.
- PostgreSQL как основную базу данных.
- SQLAlchemy 2 async для работы с БД.
- Alembic для миграций.
- Pydantic v2 для схем и настроек.
- Celery + Redis для фоновых задач.

## Последствия

- Backend остается легким и явным.
- Админку, auth и permissions нужно проектировать внутри приложения.
- Фоновые расчеты и импорт не должны выполняться в HTTP-запросах.
