# ADR 0002: Git workflow

## Статус

Принято.

## Контекст

К проекту могут подключиться другие разработчики. Нужна понятная схема веток без
избыточной сложности классического GitFlow.

## Решение

Используем облегченную схему:

- `main` - стабильная ветка, готовая к деплою.
- `develop` - основная ветка разработки.
- короткие task-ветки создаются от `develop`.
- `release/*` создается только при необходимости стабилизации версии.

Типы task-веток:

- `feature/*`
- `fix/*`
- `hotfix/*`
- `chore/*`
- `docs/*`
- `refactor/*`
- `experiment/*`

## Последствия

- Новая работа изолируется в коротких ветках.
- Интеграция идет через `develop`.
- `main` остается стабильным.
