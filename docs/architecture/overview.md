# Архитектурный обзор

## Назначение

Foosball Rating хранит и показывает рейтинги игроков в настольный футбол.
Данные поступают из внешних источников через парсинг или API, нормализуются,
сохраняются в базе и используются для расчета рейтингов.

## Компоненты

- `frontend` - публичный интерфейс и административная панель на Vue 3 + Quasar.
- `backend` - API, доменная логика, auth, права, импорт и расчеты на FastAPI.
- `db` - PostgreSQL как основное хранилище.
- `redis` - broker для Celery, временные коды, статусы задач, легкий кеш.
- `worker` - Celery worker для фоновых задач.
- `beat` - Celery Beat для периодических задач.
- `nginx` - единая точка входа и reverse proxy.

## Основные принципы

- Backend строится как feature-first modular monolith: крупные доменные модули
  внутри одного FastAPI-приложения.
- Публичные страницы должны читать уже подготовленные данные.
- Тяжелые операции выполняются в фоне.
- Исходные импортированные данные сохраняются рядом с нормализованными сущностями.
- Auth реализуется внутри проекта, без Keycloak на старте.
- Админка проектируется под реальные workflows: импорт, пересчет, пользователи,
  роли и области доступа.
