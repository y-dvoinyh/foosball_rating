# Backend

Backend на FastAPI отвечает за API, доменную модель, импорт данных, расчет
рейтингов, auth и права доступа.

## Основные модули

Backend строится как feature-first modular monolith. Основные модули находятся в
`backend/app/modules/`:

- `auth` - пользователи, роли, scopes, токены и auth-зависимости.
- `players` - игроки и соответствия внешних участников внутренним игрокам.
- `competitions` - лиги, сезоны, соревнования, команды, матчи и сеты.
- `ratings` - рейтинговые контексты, расчет, история изменений, агрегаты
  соревнований и подготовленные read-модели.
- `imports` - источники данных, raw JSON, import batches и нормализация внешних
  данных.

Live-режим на старте не выделяется в отдельный доменный модуль: polling внешних
источников относится к `imports`, пересчет и snapshots - к `ratings`.

Простые модули могут иметь плоскую структуру `models.py`, `schemas.py`,
`repository.py`, `service.py`, `router.py`, `tasks.py`. Сложные модули, прежде
всего `ratings` и `imports`, могут дополнительно делиться на внутренние слои
`domain/`, `application/`, `infrastructure/`, `presentation/`, когда это
оправдано сложностью.

## Принципы

- HTTP-запросы не выполняют долгие расчеты.
- Импорт и пересчет рейтингов запускаются как фоновые задачи.
- Исходные данные импорта сохраняются для отладки и воспроизводимости.
- Публичное API должно быть оптимизировано под чтение.
- Изменения схемы БД оформляются миграциями Alembic.
